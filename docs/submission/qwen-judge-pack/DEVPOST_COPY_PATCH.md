# Devpost Copy Patch - Current Qwen Judge Reference Runtime

**Apply only after the merged production release passes the public UI acceptance below.**

## Data-plane description

> SignalReview runs an evidence-bound Qwen Agent Society for football match review. The public judge path pins an immutable pre-match reference packet so the workflow remains reproducible throughout judging even when a live provider is unavailable. A deterministic evidence broker preserves a 30-row Quant Passport, explicit source classes, blocked/null domains, and cryptographic snapshot binding. Four Qwen Cloud roles — Statistician, Skeptic, Upside Scout, and Orchestrator — interpret and adjudicate the same packet. The current reference publishes 15 truth-bound rows, keeps the Evidence Contract PARTIAL and Confidence Ceiling LOW, and never fills missing market, confirmed-lineup, injury, travel, or live-provider facts.

## Alibaba / Qwen description

> Qwen reasoning uses Alibaba Cloud Model Studio through a server-side OpenAI-compatible API. Model credentials remain server-side. The public open-source repository includes the isolated FastAPI Qwen runtime, tests, and an Alibaba Cloud ECS deployment manifest (`deploy.sh`). Redacted Alibaba console evidence and the public deployment implementation support the Alibaba Cloud submission requirement. A running-instance or provider-availability claim is made only when confirmed by current runtime evidence.

## Agent Society description

> The Statistician synthesizes cited evidence drivers and counter-drivers. The Skeptic challenges the same evidence and constrains confidence. The Upside Scout separates primary and alternate scenarios with observable triggers and invalidation conditions. The Orchestrator classifies every specialist finding as accepted, rejected, or unresolved and publishes the final bounded verdict.

## Trust disclosure

> Qwen does not create provider facts, scores, odds, line movement, confirmed lineups, injuries, private news, deterministic metrics, probabilities, lambda, ROI, accuracy, outcomes, or certainty. Historical observations and estimated projections remain separately classified and cannot become provider-observed evidence or raise the Low confidence ceiling. Unsupported rows remain visible and null.

## Match Board disclosure

> The Match Board contains one pinned `Qwen Judge Reference Review` with the full evidence-bound ENGAGE workflow and additional versioned official schedule identities for discovery. Schedule-only rows do not inherit the reference fixture's evidence and cannot run Qwen until a fixture-specific packet is bound. The reference is labelled as an immutable pre-match review, not as a live match.
